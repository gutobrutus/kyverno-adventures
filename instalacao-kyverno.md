## 2. **Instalação do Kyverno**

Há algumas formas de instalação do kyvernos. Será utilizada a instalação via helm.

### 2.1. **Adicionar repositório**

```shell
helm repo add kyverno https://kyverno.github.io/kyverno/
```
### 2.2. **Verificar por updates**

```shell
helm repo update
```

### 2.3. **Listar versões disponíveis**

```shell
helm search repo kyverno -l
```

### 2.4. **Instalar o kyverno em alta disponibilidade**

Conforme recomendado na documentação oficial, a instalação em ambientes produtivos deve ser feita em modo HA (High Availability).

Uma constatação prévia para entender melhor o que muda no cluster, é consultar "api-resources" do cluster antes e após a instalação do kyverno:

Consultando antes da instalação:
```shell
kubectl api-resources
```

Instalação do Kyverno:
```shell
helm install kyverno kyverno/kyverno -n kyverno --create-namespace --set replicaCount=3
```
Verificando o que foi implantando:
```shell
kubectl get all -n kyverno
```

Consultando "api-resources" para verificar o que mudou no cluster:

```shell
kubectl api-resources
```
